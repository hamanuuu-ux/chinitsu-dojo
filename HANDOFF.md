## 3. 技術仕様（重要）

### 3-1. 待ち計算エンジン（JS実装・検証済み）

**方針**: DBに正解は持たない。毎回JSで14枚の和了判定を行う。

```javascript
// 13枚 → 待ち牌リスト（1〜9の全パターン試行）
function calcWaits(tiles13) {
  const waits = [];
  for (let w = 1; w <= 9; w++) {
    const h = [...tiles13, w].sort((a,b)=>a-b);
    if (canWin(h)) waits.push(w);
  }
  return waits;
}

// 14枚のアガリ判定（完全バックトラック）
function canWin(h) {
  const c = new Array(10).fill(0);
  for (const t of h) c[t]++;
  const tried = new Set();
  for (const t of h) {
    if (tried.has(t)) continue;
    tried.add(t);
    if (c[t] >= 2) {
      c[t] -= 2;
      if (solveM(c, 4)) { c[t] += 2; return true; }
      c[t] += 2;
    }
  }
  return false;
}

// 残り面子をカウント配列から取り出せるか再帰探索
function solveM(c, rem) {
  if (rem === 0) { for(let i=1;i<=9;i++) if(c[i]!==0) return false; return true; }
  let n = -1;
  for (let i=1;i<=9;i++) { if(c[i]>0){n=i;break;} }
  if (n < 0) return false;
  // 刻子
  if (c[n] >= 3) { c[n]-=3; if(solveM(c,rem-1)){c[n]+=3;return true;} c[n]+=3; }
  // 順子
  if (n<=7 && c[n]>=1 && c[n+1]>=1 && c[n+2]>=1) {
    c[n]--;c[n+1]--;c[n+2]--;
    if(solveM(c,rem-1)){c[n]++;c[n+1]++;c[n+2]++;return true;}
    c[n]++;c[n+1]++;c[n+2]++;
  }
  return false;
}
```

**検証**: Python（同一ロジック）で全49問 × 9パターン（1〜9萬を加えた14枚）= 441パターンを厳密バックトラックと照合済み。完全一致確認。

### 3-4. スコア計算式

```
問題正解時のスコア加算 = max(基準時間 - 実解答秒数, 0)
基準時間 = 60秒（全難易度統一）
不正解（見逃し・誤選択含む）= 0点
```

## 5. 設計上の決定事項（変更禁止）

| 決定事項 | 理由 |
|----------|------|
| **難易度分類を表示しない** | メタ読み（難易度から待ち数を推測）防止 |
| **全パターン網羅はしない** | 物理的に不可能 |
| **待ちはDBに持たず毎回計算** | DB誤りによる誤判定を根本排除 |
| **同一問題内でスーツを混在させない** | チンイツのルール上あり得ない |
| **不正解時はスコア0（マイナスなし）** | 体験設計の方針 |

### 優先タスク（v1完成に向けて）

**①ネイティブアプリ化**
- 推奨: Flutter（iOS/Android両対応、単一コードベース）
- 代替: React Native
- テスト版HTMLのロジックをそのまま移植可能

**②問題DBの拡充**
- 現在の49問を200問以上に拡充
- 生成方法: テンパイ形を手続き的に生成し、`calcWaits()`で待ち確認
- 手牌生成の条件:
  - 13枚すべて同一スーツ（1〜9の数牌）
  - 少なくとも1種類以上の待ちがある（テンパイ確認）
  - 重複排除（ソート後の一致チェック）

**③お気に入り・苦手問題機能**
```
お気に入り: 任意タイミングで登録。正解後も残る。手動削除。
苦手問題:  誤答時に自動登録。正解で自動削除。
          ただしお気に入りと重複する場合はお気に入り優先（削除しない）
```

### コード移植の注意点

1. **待ち計算エンジンは必ずそのまま使う**（Section 3-1のアルゴリズム）  
   変更するとバグが入る可能性が高い。Pythonで検証済みの実装を信頼すること。

2. **牌画像のキー命名規則**  
   `{suit_char}{num}` → `m1`〜`m9`, `p1`〜`p9`, `s1`〜`s9`

3. **スーツのランダム選択は問題単位**  
   `Math.random()`で0〜2のインデックスを取得し、問題全体に同一スーツを適用

4. **手牌表示は必ずソート順**  
   `tiles.sort((a,b)=>a-b)` でソートしてから表示

## 9. スコア・レーティング設計（v2向け）

### スコア式（対戦用・確定版）
```
問題スコア = max(基準時間 - 実解答秒数, 0)
基準時間（難易度別）:
  初級（3面以下の待ち）: 30秒
  中級（4〜6面待ち）  : 45秒
  上級（7面以上待ち） : 60秒
対戦スコア = Σ（正解した問題のスコア）
勝敗: スコアの高い方が勝利（同点は引き分け）
```
※ テスト版では難易度非表示のため全問60秒で統一

### ELOレーティング
```
初期レーティング: 1000
勝利: +15〜+30（相手レーティングが高いほど大きい）
敗北: -10〜-25（相手レーティングが低いほど大きい）
引き分け: ±0
K係数: 50（200戦まで）→ 30（以降）
```
